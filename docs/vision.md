# SwallowLoop · 燕子回环：项目设想

> **一句话概念**
> **SwallowLoop · 燕子回环：围绕你的代码仓，通过多阶段流水线实现 AI 辅助开发，每一阶段由人类审批把关。**

## 一、项目愿景

- 为**程序员个人 / 小团队**提供一个** Human-in-the-loop 的开发流水线系统**：
  只需创建 Issue 并审批 AI 在每个阶段的产出，就能高效地完成代码开发。
- 重点服务 **单代码仓库、中大型代码库** 的日常维护与演进，包括：
  - 功能实现
  - Bug 修复
  - 技术设计
  - 代码提交

## 二、核心设计思想

1. **Human-in-the-loop**
   - 每个阶段由 AI 产出 Markdown 文档，人类审批通过后才进入下一阶段
   - 人类保留最终决策权，AI 是助手而非替代者

2. **7 阶段流水线**
   - 头脑风暴 → 方案成型 → 详细设计 → 任务拆分 → 执行 → 更新文档 → 提交
   - 每阶段都有明确的输入、输出和审核点

3. **透明可控**
   - 所有 AI 产出都存储为 Markdown 文档
   - 每次审批记录在案，形成完整的审核历史

4. **闭环追溯**
   - 每个 Issue 从创建到提交形成可追踪闭环
   - 废弃的 Issue 3 天后自动删除

## 三、目标使用体验

对用户（开发者 / 仓库维护者）来说，理想体验是：

1. 在 Web 前端创建一个 Issue（标题 + 描述）。
2. SwallowLoop 自动进入**头脑风暴**阶段，生成多个可行方案。
3. 你选择一个方案，审批通过。
4. AI 自动进入**方案成型**阶段，产出整体实现思路。
5. 你审核思路，通过后继续。
6. 重复上述过程：详细设计 → 任务拆分 → 执行 → 更新文档 → 提交 PR。
7. 你只需在关键节点审批，AI 完成大部分工作。

简单说：**「人类做决策与审批，燕子帮你干活。」**

## 四、当前功能状态

| 功能 | 状态 | 说明 |
|-----|------|-----|
| Issue 管理 | ✅ | CRUD 操作 |
| 7 阶段流水线 | ✅ | 完整的状态机 |
| 阶段审批 | ✅ | 通过/打回操作 |
| 阶段状态追踪 | ✅ | pending/approved/rejected/running/error |
| TODO 列表 | ✅ | 执行阶段的任务拆分 |
| 进度追踪 | ✅ | 百分比 + executionState |
| 评论历史 | ✅ | 审核意见记录 |
| Issue 归档/废弃 | ✅ | 生命周期管理 |
| Web Dashboard | ✅ | 状态展示 + 实时日志 |
| 自更新 | ✅ | 检测新版本自动更新 |

## 五、后续演进方向

- **AI 执行集成**
  - 集成 IFlow Agent 执行实际代码任务
  - 根据 TODO 列表循环执行

- **PR 创建集成**
  - 在 submit 阶段自动创建 PR
  - 支持 GitHub PR 操作

- **多项目支持**
  - 支持同时管理多个代码仓库
  - 项目级别的配置隔离
